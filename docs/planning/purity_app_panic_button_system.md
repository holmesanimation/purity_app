## Core Vision

The Panic Button system is not merely a browser blocker or anti-porn mechanism.

It is intended to function as:

- an emergency interruption system,
- a spiritual/emotional triage workflow,
- a behavioral pattern awareness system,
- and a recovery-oriented intervention experience.

The goal is:
- interrupt impulsive behavior,
- identify the underlying heart/emotional state,
- encourage honesty and prayer,
- create physical disengagement from the computer,
- promote accountability,
- and gradually build self-awareness over time.

---

# Persistent Panic Button

## Desktop Presence

The panic button always exists on screen.

Placement:
- bottom-right of monitor
- above the system tray

Appearance:
- small red circular button
- semi-transparent by default (~40–50%)
- fully opaque on hover
- always-on-top
- subtle/glass-like appearance

Behavior:
- visible but not visually overwhelming
- psychologically available at all times
- intended to function like an emergency escape mechanism

Future possibilities:
- optional subtle pulse/glow during elevated risk states
- draggable only in settings mode

---

# Danger Zone Escalation

## Elevated Risk Awareness

The browser extension already restricts browsing to approved/whitelisted sites.

When the user attempts to access a non-whitelisted URL:
- the extension asks whether to whitelist the new URL

If this occurs:
- the system enters an elevated danger/risk state.

### Panic Button Elevated State

Changes:
- grows by ~25%
- becomes fully opaque
- optional subtle pulse/glow animation

Purpose:
- not punitive
- not shaming

Instead:
- visually communicates:
  "You are drifting into danger territory."

When browsing session ends:
- button returns to normal size
- transparency restored
- danger state cleared

# Panic Button Pressed

## Immediate Interruption

The moment Panic is pressed:

Actions:
- immediately close browser
- optionally kill browser processes entirely
- optionally force focus to intervention dialog

Purpose:
- interrupt temptation momentum immediately
- prevent continued escalation
- remove access to triggering content

This is intentionally an emergency "circuit breaker."

# Countdown Recovery Phase

## Trigger

ONLY begins after:
- all sticky notes have been acknowledged

At this point:
- sticky notes become read-only
- browser remains closed
- no immediate browser restoration

---

## Purpose

The purpose of the countdown phase is:

- physical disengagement from the computer
- interruption of dopamine/impulse cycle
- calming emotional escalation
- encouraging prayer and movement

The user should NOT remain seated at the keyboard.

# Repeated Panic Escalation

## Automatic Accountability Escalation

If Panic Button is pressed repeatedly within a configurable time window:

Then:
- notification is automatically sent to accountability group

This behavior:
- configurable in settings
- DEFAULT = enabled

Purpose:
- repeated panic events likely indicate escalating struggle
- prevent suffering in isolation
- encourage intervention/support

Possible future escalation logic:

### Calm
- default button
- semi-transparent

### Elevated
Triggered by:
- whitelist overrides
- risky browsing behavior
- prolonged sessions
- late-night browsing

Effects:
- enlarged panic button
- opaque state
- subtle pulse

### Critical
Triggered by:
- repeated panic sessions
- repeated overrides/escalations

Effects:
- automatic accountability notification
- stronger intervention behaviors

# Core Design Philosophy

This system is evolving beyond:
> "anti-porn software"

Into something closer to:
> an interactive spiritual recovery and accountability workflow.

The core sequence is:

1. Interrupt
2. Identify causes
3. Reflect honestly
4. Engage spiritually
5. Physically disengage
6. Cool down
7. Encourage accountability
8. Build long-term self-awareness
9. Provide continued gentle support
